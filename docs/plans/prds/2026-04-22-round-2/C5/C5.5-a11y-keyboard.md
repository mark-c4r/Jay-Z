### Check 5: Branch A + B smoke (integration)

### Check 6: Clamping (sanity)

In devtools console:

```javascript
[...document.querySelectorAll('.yt-facade')].map(f => {
  const fs = +f.dataset.start;
  const seeks = [...f.nextElementSibling?.querySelectorAll('[data-seek]') || []].map(b => +b.dataset.seek);
  return { facadeStart: fs, minSeek: Math.min(...seeks) };
});
```

For every facade, `minSeek >= facadeStart`. No button should seek before its facade window begins.

## Known risks + mitigations

1. **CLS edge-case on slow connection** — if chapter UL renders slightly after facade layout, there may be a brief CLS spike. Mitigation: `min-height: 80px` on `.video-chapters` reserves space eagerly. If still flagged, Codex suggests moving `renderVideoChapters()` earlier in IIFE.
2. **Focus-visible variance across browsers** — Firefox uses keyboard-initiated focus for `:focus-visible`; Chrome requires the `:focus-visible` pseudo to be supported (shipped Chrome 86+). No issue with target browsers.
3. **Manual checks (2, 3, 5)** — Sonnet can perform Check 2 + 5 via Chrome devtools; Check 3 (Lighthouse) typically requires operator-run (Lighthouse API can run from node but is out of typical PRD automation scope). If operator runs manually, record results in the commit body.
