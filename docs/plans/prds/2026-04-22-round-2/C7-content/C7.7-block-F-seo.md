## Known risks + mitigations

1. **Prompt quality drift** — some prompts may not produce great output in practice. Mitigation: the reader can iterate; each prompt is a starter, not a final spec. Block F's intro sets this expectation.
2. **`<pre>` block rendering on narrow screens** — long lines wrap but look awkward. `.recipe-prompt` uses `white-space: pre-wrap; word-wrap: break-word` to prevent horizontal scroll.
3. **HTML-entity encoding in prompts** — prompts contain `<`, `>` characters (e.g. `&lt;script type="application/ld+json"&gt;` in recipe 5). These must stay HTML-escaped inside the `<pre>` so the text renders literally, not as embedded markup.
4. **Word count might exceed 1700** — current draft is ~1500. Comfortable room.
